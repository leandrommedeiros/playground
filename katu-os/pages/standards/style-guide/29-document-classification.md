# 29. Document Classification

Official KatuOS documentation is divided into two categories:

- Technical Manuals
- Supporting Documents

## Technical Manuals

Technical Manuals document permanent engineering knowledge and should adopt the
standard KatuOS cover page.

This category includes documents such as:

- Scheduling, Delays & Synchronization Manual
- Kernel Architecture Manual
- Documentation Style Guide
- API Reference

These documents typically contain multiple chapters, architectural discussions
and long-term technical reference material.

## Supporting Documents

Supporting Documents complement the documentation library but are not intended
to serve as comprehensive technical manuals.

They may adopt a simplified document header while preserving the editorial
identity of the project.

Typical examples include:

- README.md
- Documentation Guide
- CONTRIBUTING.md
- Release Notes
- Handover Documents

A simplified header normally consists of:

- Document title
- Revision identifier
- Optional KatuOS logo

The objective is to preserve visual consistency while avoiding unnecessary
cover pages for short documents.

Regardless of their classification, all official documents shall follow the
editorial principles defined throughout this guide.
